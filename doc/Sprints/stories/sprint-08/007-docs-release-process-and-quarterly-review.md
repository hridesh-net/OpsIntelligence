# STORY-049 — Docs release process and quarterly review

| Field | Value |
|-------|--------|
| **Sprint** | sprint-08 |
| **Type** | Process |
| **Priority** | P2 |
| **Estimate** | S |

## Summary

Define **release documentation process**: every PR with user-visible change updates docs/CHANGELOG; **quarterly** review of top 10 pages; owner rotation.

## User story

**As a** tech writer / maintainer**  
**I want** a lightweight process**  
**So that** docs stay accurate.

## Acceptance criteria

1. **Checklist** in CONTRIBUTING for PR authors.
2. **Quarterly** calendar reminder + template.
3. **Metrics**: top doc pages from analytics (if any) or manual survey.
4. **Retro** notes template for doc improvements.
5. **Ownership** table in `doc/README.md`.

## Definition of Done

- [ ] First quarterly review scheduled before sprint end.

## Dependencies

- STORY-048.

## Risks

- Process ignored; tie to release checklist.
